    存储引擎

    1. MyiSAM 存储引擎#

    1.1 特性：#

    • 支持表级锁：对表的读写操作都会对表进行加锁。
    • 并发性：对于表的插入和修改操作，并发性不太好。对读的操作支持的比较好。
    • 支持对表的埙坏进行修复。注意：对于表的修复，可能造成数据的丢失
    • 支持全文索引
    • 支持数据压缩 。 myisampack命令进行压缩

    1.2 限制#

    • 版本 < 5.0 表的默认大小为4g
    • 版本 > 5.0 表的默认大小为256TB
    • 不支持事务

    1.3 使用场景#

    • 非事务应用
    • 只读类应用

    2. innodb 存储引擎#

    • 版本5.5之后的默认支持存储引擎
    • 它是事务型引擎，完全支持事务的acid特性
    • 支持行级锁 （锁： 共享锁和独占锁） 共享锁：读操作。 独占锁：写操作
    • 行级锁可以最大程度的支持并发
    • 使用表空间进行数据存储
      • 系统表空间： 系统表空间无法简单的收缩文件大小。存在IO瓶颈
      • 独立表空间： 可以通过optimize table 命令来收缩系统文件 可以同时向多个文件刷新数据。
      • 建议使用独立表空间
